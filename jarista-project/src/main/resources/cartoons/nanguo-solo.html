Where 南郭s Hide in Software

Vague Specs

// "Improve performance"
// "Make it better"
// "Handle edge cases"

Specs without LoadBearing status are organizational — they can't hide unfinished work because they don't claim any. A leaf must declare: PLANNED, IN_PROGRESS, or DONE.

Status Theater

status: "on track" ← who says?
progress: 73% ← of what?
risk: "low" ← by whose measure?

Jarista's status is boolean at the piece level (done or not) and derived at the organizational level. No subjective percentages. No curated dashboards.

The Graph Solo

spec: LoadBearing → status()
piece: done() true | false
validator: runs every build

The graph validator is King Min. It doesn't fire anyone — it simply asks each node to play solo. The structure reveals truth without assigning blame.
